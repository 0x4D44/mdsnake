# HLD — Coil: a snake puzzle game

**Date:** 2026-06-20
**Status:** vertical-slice scaffold built; design baseline for iteration.

## 1. Concept

You are a snake; you solve puzzles by exploiting snake-specific abilities. The
snake's **body is the tool** — the thing no other puzzle avatar has. Progression
runs from a short snake doing simple things (strike to grab a door handle) up
through length, cooperation, and exotic senses.

Differentiator vs. prior art: Snakebird already nails "body + gravity" as pure
puzzle; Snake Pass does physical 3D slither. Neither builds a **curriculum of
species abilities** (strike, growth, constriction, rattle, heat-sense,
swallow-and-carry, co-op). That ladder is the original spine.

## 2. Decisions taken

| # | Decision | Rationale |
|---|----------|-----------|
| D1 | **Discrete, turn-based** (not physics slither) | Keeps focus on thinking; undo/par/determinism become free. |
| D2 | **2.5D + gravity** — puzzle in a vertical plane (x, y), rendered in orbitable 3D; depth axis reserved for occasional later puzzles | Humans can't reason about full-3D body knots ("hold it in your head"). |
| D3 | **Browser document**, built like `night-cab` | Standalone Vite/TS/Three.js source repo; `dist/` copied into the almanac as a `<slug>/`. Best dev experience *and* catalog placement. |
| D4 | **Pure, tested sim core** separate from renderer | Determinism makes the rules an oracle; undo is a snapshot stack. Matches the testing-oracle quality philosophy. |
| D5 | **Score-light**: no timers/lives on the main path | Puzzle reward is the solve; scoring goes into optional layers (eggs, move-par). |

## 3. Core rules (v1, as implemented)

- World is a sparse grid: only `wall`/`fruit`/`exit` cells exist; everything else
  is air. Gravity pulls toward −y.
- The snake is an ordered list of segments (head first), fixed length unless it
  eats.
- **move(dir):** head steps one cell (blocked by wall / own body, but may enter
  the tail cell it vacates; entering fruit grows it; entering exit wins). Then
  **gravity settles**.
- **strike(dir):** the head launches up to `strikeRange` cells with gravity
  suppressed mid-flight, then a single settle. This is the "jump" — it sails over
  gaps and reaches ledges a plain move can't. *(Future: range scales with how
  much body you coil to charge.)*
- **Gravity:** only walls bear weight; segments don't support each other, so the
  whole snake falls as one rigid body until supported, or below `floorY` → death.
- Win = head on exit. Death = void. Both reset cheaply.

All rules are pure functions in `src/core/game.ts`; covered by `game.test.ts`
(move/trail, wall block, self block, tail-vacate, eat/grow, exit win, gravity
settle, void death, strike-over-gap).

## 4. Ability toolbox (the progression spine)

Body-as-structure · strike/coil · length/growth · wrap/constrict · **rattle**
(sound triggers) · **heat-sense** (dark) · **swallow & carry** · shed-skin decoy ·
**co-op/relay**. Length is the master resource: longer = more reach and more
plates held, but easier to trap yourself.

Worlds introduce one ability each, then a "twist" room that breaks the
assumption, then a finale that recombines: Hatchling → Growth → The Climb →
Rattle → Dark → The Gullet → Two Snakes → Finale.

## 5. Scoring

- **Eggs** (the stars): 1 solve, 1 hidden, 1 optional-constraint (e.g. "don't
  grow past N"). Loosely gate later worlds so a stuck player still progresses.
- **Move-par** shown *after* the first solve (never before).
- Generous instant **undo + restart**; no timers, no lives.

## 6. Vertical slice

Room 1 teaches the v1 vocabulary in one go: walk → eat (grow) → strike across a
void → reach exit. If move+strike+grow under gravity feels good, the rest of the
toolbox is content on the same engine.
